# 아이데이션 — 마이크로팩토리: 맥주 제조

- 목적: 손안에서 볼 수 있는 제조 공정 시각화, 단계별 의사결정 지원, 상용화 경로 탐색
- 대상: 개인 브루어리(홈브루/스몰 배치), 기존 상용 레퍼런스를 참고해 변형 레시피 설계
- 핵심가치: 즉시성(모바일 우선), 투명성(단계/자원/시간 가시화), 시뮬레이션(What‑if)
- 주요 화면: 공정 맵(DAG/타임라인), 단계 상세(입력/출력/파라미터), 상용화 캔버스(원가/규제/유통), 레퍼런스 비교
- 인터랙션: 단계 선택→파라미터 조절(온도/시간/비중 등)→출력 변화 미리보기→플로우 저장/공유

## 맥주 공정 단계(초안)
1. 원료 준비·분쇄
2. 매싱(온도·시간)
3. 여과(라우터링)
4. 보일·홉 투입(IBU)
5. 워트 냉각
6. 발효(효모/온도/시간, OG/FG, ABV)
7. 숙성(컨디셔닝)
8. 여과/탄산화(SRM/탄산 레벨)
9. 패키징(병/캔/케그)

## 상용화 연결(초안)
- 레퍼런스 스타일 선택→레시피 변형 가이드(APA, IPA, 라거 등)
- BOM·원가 산출(배치 크기/손실률/효율)
- 설비 제약(탱크 용량/열원), 규제 체크리스트(지역별 라벨링/주세)
- 유통 경로 시나리오(직판/리테일/온·오프프)

## 데이터 모델(가벼운 스케치)
- Step(id, name, inputs, outputs, params, duration)
- Recipe(id, name, style, steps[])
- ReferenceProduct(name, style, metrics{IBU, SRM, ABV})

## 미해소 불확실성
- 실시간 센서 연동 범위 vs 정적 시뮬레이션
- 법규/인허가 범위(지역 스코프)
- 레퍼런스 데이터 소스·라이선스
- 정확도(과학 모델)와 UX 단순화의 균형

## 첫 실행 단위(70% 수준)
- 정적 데모: 공정 맵 + 단계 상세 패널 + 기초 원가 계산
- 샘플 레시피: American Pale Ale, Pilsner 비교

## 랜딩 페이지 시각화 (Scrollytelling)
*목적: 단순한 검은색 화면을 탈피하고, 진입 즉시 "맥주 제조 경험"을 직관적으로 전달하며 서비스의 가치를 스크롤에 따라 스토리텔링한다.*

- **시각적 메타포 (Beer Metaphor)**
  - 배경: 짙은 앰버(Amber) 색상의 액체 질감과 하단에서 올라오는 기포(Bubbles) 애니메이션.
  - 상단: 부드러운 거품(Foam) 형태의 헤더 또는 트랜지션.
  - 톤앤매너: 다크 테마를 유지하되, 조명(골드/오렌지 빛)을 받아 빛나는 맥주잔 내부의 느낌 연출.

- **스크롤 반응형 스토리보드 (What you can get)**
  - **섹션 1: Intro (맥주 잔 속으로)**
    - "손 안의 마이크로팩토리, 당신만의 맥주를 설계하세요."
    - 시각: 화면 전체가 맥주로 채워지며 기포가 올라오는 몰입형 애니메이션.
  - **섹션 2: Ingredients & Process (9단계 공정 가시화)**
    - "물, 몰트, 홉, 효모. 그리고 9단계의 정교한 공정."
    - 시각: 스크롤 시 위에서 재료(아이콘)가 떨어지고, 화면이 끓어오르는(Boiling) 연출로 전환.
  - **섹션 3: Simulation (나만의 레시피)**
    - "온도 1도, 홉 1그램이 만드는 풍미의 차이를 미리 확인하세요."
    - 시각: 온도계와 게이지(IBU, ABV)가 스크롤에 맞춰 동적으로 움직임.
  - **섹션 4: Commercialization (상용화의 길)**
    - "레시피를 넘어, 브루어리 창업을 위한 원가와 패키징까지."
    - 시각: 완성된 맥주가 캔/병으로 포장되어 레일 위를 지나가는 연출.
  - **섹션 5: Call to Action (시뮬레이터 진입)**
    - 스크롤의 끝에서 거품이 걷히며 현재 작업 중인 "시뮬레이터 대시보드"가 부드럽게 등장(Fade-in).

## 시뮬레이터 고도화 (Simulator Dashboard)
*목적: 사용자가 현재 공정의 위치를 명확히 인지하고, 파라미터 변경 시 시판 레퍼런스와의 차이를 직관적으로 비교할 수 있도록 한다.*

- **공정 파이프라인 시각화 (현재 위치 인지)**
  - 단순 목록이 아닌 **가로형 타임라인 또는 연결된 노드(DAG)** 형태로 9단계를 시각화.
  - 현재 선택된 단계(Active Step)와 완료된 단계, 진행해야 할 단계를 색상/아이콘으로 구분하여 "내가 지금 공정의 어느 위치에 있는지" 즉각적으로 인지.

- **레퍼런스 비교 인디케이터 (What-If 분석)**
  - **시판 맥주 비교**: 파라미터 조절 시, 유명 상업용 American Pale Ale(예: Sierra Nevada Pale Ale 스타일)의 표준 데이터(온도, IBU, ABV 등)를 옅은 선이나 마커로 함께 표시.
  - **표준 범위(Style Guideline) 표시**: APA 스타일의 적정 가이드라인(예: IBU 30~50, ABV 4.5~6.2%)을 게이지 또는 슬라이더 배경에 색상 밴드로 표시.
  - **결과 즉시 반영**: 내가 입력한 값(내 레시피)이 표준 범위 내에 있는지, 시판 레퍼런스보다 쓴지/단지/무거운지를 방사형 차트나 바 게이지 형태로 즉시 피드백.

- **단계별 인터랙션**
  - 파라미터 변경 시, 해당 단계의 예상 결과물(워트의 색상, 예상 IBU 등)을 실시간으로 업데이트하여 보여줌.

## 글로벌 레퍼런스 & 인터랙티브 탐색 (Global Discovery & Liquid Realism)
*목적: 세계의 대표 맥주를 탐험하는 재미를 주고, 액체(맥주)의 물리적 속성을 시각적으로 정교하게 표현하여 몰입감을 극대화한다.*

- **지구본(Globe) 기반의 레퍼런스 선택기**
  - 단순한 드롭다운(`<select>`) 대신, **3D 지구본 또는 인터랙티브 세계 지도**를 상단에 배치.
  - 세계 판매량 및 인지도 기준 15개국 대표 맥주(예: 기네스, 하이네켄, 칭따오, 아사히 등)가 지도 위에 마커로 표시됨.
  - 마커를 클릭하면 해당 맥주의 기본 스펙(ABV, IBU, 색상 등)이 로드되며 "이 맥주를 베이스로 만들기"가 시작됨.

- **맥주의 시각적 리얼리즘 강화 (투명도 및 SRM)**
  - 기존 비커 애니메이션에 **투명도(Clarity/Turbidity)** 지표를 추가.
  - 여과(라우터링) 단계를 거치면 탁했던 액체가 맑아지고, 밀맥주(Wheat)나 Hazy IPA 스타일을 선택하면 불투명한 상태가 유지되도록 CSS `backdrop-filter`나 `opacity`를 동적으로 조절.
  - 맥주 색상(SRM) 지표를 추가하여, 로스팅된 맥아 비율이나 끓임 시간에 따라 밝은 황금색에서 짙은 흑흑색(Stout)까지 액체 색상이 부드럽게 변하도록 구현.
